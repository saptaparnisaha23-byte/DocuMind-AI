# ADR-001: Selection of ChromaDB as the Vector Database

## Context

DocuMind-AI requires a vector database to store document embeddings and perform semantic similarity searches efficiently. The system must support local development, integrate well with Python, and be lightweight enough for a student project while remaining scalable for future improvements.

## Decision

We selected **ChromaDB** as the vector database.

ChromaDB integrates easily with Python, supports persistent local storage, and provides efficient similarity search over embeddings generated by Sentence Transformers.

## Consequences

### Positive

- Easy Python integration
- Persistent local storage
- Lightweight
- No external server required
- Good documentation

### Negative

- Limited enterprise features compared to Pinecone or Weaviate
- Not intended for very large production datasets

## Alternatives Considered

### Pinecone

Rejected because it requires a cloud account and usage limits.

### FAISS

Rejected because it does not provide persistent storage as conveniently as ChromaDB.

### Weaviate

Rejected because it introduces additional deployment complexity for this internship project.
